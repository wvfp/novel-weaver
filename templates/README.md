# Obsidian 工作流集成说明

本文档介绍如何将 novel-weaver 的模板集成到 Obsidian 中使用，帮助你高效管理小说创作。

---

## 模板文件说明

本目录包含三个 Obsidian 模板文件：

| 文件 | 用途 | 生成的文件名规则 |
|------|------|------------------|
| `obsidian-chapter-template.md` | 小说章节模板 | `ch{章节号}-{标题}.md` |
| `obsidian-dungeon-template.md` | 副本设定模板 | `dungeon-{名称}.md` |
| `obsidian-world-template.md` | 世界观设定模板 | `world-{名称}.md` |

---

## 安装步骤

### 1. 打开 vault

在 Obsidian 中，将 `.novel-weaver/content/` 目录作为 vault 打开。

### 2. 安装 Templates 插件

1. 进入 **设置 → 社区插件 → 浏览**
2. 搜索并安装 **Templates** 插件（由 Obsidian 官方提供）
3. 启用该插件

### 3. 设置模板文件夹

1. 进入 **设置 → Templates**
2. 将 **模板文件夹位置** 设置为包含上述 `.md` 文件的 `templates/` 目录
3. 确认路径正确

### 4. 使用模板

在编辑器中：

- **Windows / Linux**：按 `Ctrl + T`
- **macOS**：按 `Cmd + T`

在弹出的模板列表中选择需要的模板即可自动插入。

## Wikilink 命名规范

为了保持关联引用的一致性，请遵循以下命名规则：

### 角色

- 显示名称：`[[角色名]]`
- 文件名：`char-{名称}.md`
- 示例：`[[林墨]]` → 文件 `char-林墨.md`

### 副本

- 显示名称：`[[副本名]]`
- 文件名：`dungeon-{名称}.md`
- 示例：`[[迷雾森林]]` → 文件 `dungeon-迷雾森林.md`

### 章节

- 显示名称：`[[章节标题]]`
- 文件名：`ch{章节号}-{标题}.md`
- 示例：`ch01-初入异世界.md`

### 世界观 / 设定

- 显示名称：`[[世界名]]`
- 文件名：`world-{名称}.md`
- 示例：`[[天元大陆]]` → 文件 `world-天元大陆.md`

## YAML 前置元数据字段说明

每个模板文件头部包含 `---` 包裹的 YAML 前置元数据，以下是各字段的说明：

### 通用字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 标题 | `"初入异世界"` |
| `type` | 文档类型：`chapter` / `dungeon` / `world` | `chapter` |
| `status` | 状态：`draft` / `active` / `review` / `published` | `draft` |
| `created` | 创建日期（自动生成） | `2026-05-31` |
| `tags` | 标签列表 | `[chapter, 异世界]` |

### 章节专属字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `chapter_num` | 章节编号 | `1` |
| `dungeon_id` | 关联副本的 ID | `dungeon-迷雾森林` |
| `word_count` | 字数统计 | `2500` |

### 副本专属字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `dungeon_id` | 副本唯一标识 | `dungeon-迷雾森林` |
| `difficulty` | 难度等级：`easy` / `medium` / `hard` / `boss` | `hard` |
| `theme` | 副本主题 | `森林` |

## Graph View 使用技巧

Obsidian 的图谱视图（Graph View）是可视化故事关系的强大工具：

### 基础操作

1. 点击左侧栏的 **图谱视图** 图标（圆点连线）
2. 使用鼠标滚轮缩放
3. 拖动节点查看更多关联

### 按类型过滤

在图谱视图中，通过颜色或标签筛选不同类型节点：

- **章节**（`type: chapter`）— 故事情节节点
- **副本**（`type: dungeon`）— 冲突与挑战节点
- **世界观**（`type: world`）— 背景设定节点
- **角色** — 人物关系节点

### 关系追踪

- 点击任意节点查看其双向链接
- 使用 **局部图谱**（Local Graph）查看当前文件的直接关联
- 开放的链接（未创建的文件）会显示为虚线圆圈 — 双击即可创建

### 创作规划建议

1. 先创建世界观和副本节点，建立故事骨架
2. 创建角色节点，关联到相应副本和世界观
3. 按顺序创建章节节点，用 wikilink 关联到相关副本和角色
4. 通过图谱视图检查故事脉络是否完整，发现遗漏的关联

---

## 注意事项

- 模板中的 `{{变量}}` 在使用时会保留，需要手动替换为实际内容
- 建议先创建世界观和副本模板，再创建章节模板，确保链接完整
- 如果修改了模板文件，需要在 Obsidian 中重新加载插件才能生效
- 所有文件建议保存在 `.novel-weaver/content/` 目录下，便于 novel-weaver 工具链处理
