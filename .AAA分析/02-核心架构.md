# Novel Weaver 核心架构

## 架构概览

Novel Weaver 是一个基于 OpenCode 插件 SDK 的 AI 辅助小说创作系统，采用插件化架构设计。

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenCode 平台                             │
│              (提供 LLM、会话、工具调用能力)                    │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       │ 插件接口 (@opencode-ai/plugin)
                       ▼
┌─────────────────────────────────────────────────────────────┐
│              Novel Weaver 插件 (novel-weaver)                │
├─────────────────────────────────────────────────────────────┤
│  工具层 (Tools)                                              │
│  ├── novel_init (项目初始化)                                  │
│  ├── novel_world_* (世界观管理)                               │
│  ├── novel_dungeon_* (副本管理)                               │
│  ├── novel_character_* (角色管理)                             │
│  ├── novel_write_* (章节写作)                                 │
│  ├── novel_review_* (质量审查)                                │
│  ├── novel_consistency_* (一致性检查)                         │
│  ├── novel_progress_* (进度追踪)                              │
│  ├── novel_pipeline_* (写作管线)                              │
│  └── novel_query/stats/ping (查询统计)                        │
├─────────────────────────────────────────────────────────────┤
│  Agent 层                                                     │
│  ├── World Builder (世界观构建师)                              │
│  ├── Dungeon Master (副本主神)                                │
│  ├── Reviewer (网文审查员)                                    │
│  └── Plot Planner (大纲规划师)                                │
├─────────────────────────────────────────────────────────────┤
│  模块层 (Modules)                                             │
│  ├── Chapter Engine (章节引擎)                                │
│  ├── Review System (审查系统)                                 │
│  ├── Consistency Check (一致性检查)                           │
│  ├── RAG (检索增强生成)                                       │
│  ├── Style Anchor (风格锚点)                                  │
│  ├── Style Imprint (风格印记)                                 │
│  ├── Summary Engine (摘要引擎)                                │
│  ├── Crosscheck (交叉验证)                                    │
│  ├── Foreshadow (伏笔追踪)                                    │
│  ├── State Snapshot (状态快照)                                │
│  └── Annotations (注释系统)                                   │
├─────────────────────────────────────────────────────────────┤
│  数据层                                                       │
│  ├── sql.js (WASM SQLite)                                    │
│  ├── Markdown 文件存储                                        │
│  └── Vector Store (向量存储)                                  │
├─────────────────────────────────────────────────────────────┤
│  Dashboard (Web 面板)                                         │
│  ├── Express Server                                          │
│  ├── API Routes                                              │
│  └── Web UI                                                  │
└─────────────────────────────────────────────────────────────┘
```

## 插件入口 (src/index.ts)

插件入口是 Novel Weaver 与 OpenCode 平台的连接点。

```typescript
// 插件生命周期
export default definePlugin({
  // 1. 初始化
  async init(context) {
    // 加载配置
    // 初始化数据库
    // 注册工具
  },
  
  // 2. 工具注册 (23个工具)
  tools: [
    novel_init,
    novel_world_create,
    novel_world_query,
    // ... 更多工具
  ],
  
  // 3. Agent 注册 (4个Agent)
  agents: [
    worldBuilderAgent,
    dungeonMasterAgent,
    reviewerAgent,
    plotPlannerAgent,
  ],
  
  // 4. 聊天参数
  chat: {
    params: {
      temperature: getTemperatureForAgent(agent),
    },
  },
});
```

## 数据库层 (src/db/)

### 数据库架构

Novel Weaver 使用 sql.js (WASM SQLite) 作为数据库引擎，所有操作同步执行。

#### 核心表 (13张)

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `projects` | 小说项目 | name, genre, pipeline_phase |
| `worlds` | 世界观设定 | name, type, description, parent_id |
| `characters` | 角色数据 | name, status, attributes |
| `dungeons` | 副本数据 | name, theme, difficulty, world_id |
| `chapters` | 章节数据 | title, num, volume, dungeon_id, status |
| `reviews` | 审查记录 | chapter_id, score, issues |
| `links` | 实体关联 | source_id, target_id, relation |
| `progress` | 攻略步骤 | dungeon_id, step, status |
| `schema_version` | 迁移版本 | version |
| `chapter_facts` | 章节事实 | chapter_id, fact_type, content |
| `character_states` | 角色状态 | character_id, chapter_id, state |
| `outlines` | 大纲层级 | type, parent_id, content |
| `aliases` | 实体别名 | entity_id, alias, entity_type |

#### FTS4 全文搜索索引 (3个)

| 索引 | 覆盖表 |
|------|--------|
| `chapters_fts` | chapters |
| `worlds_fts` | worlds |
| `characters_fts` | characters |

#### 数据库初始化流程

```
1. 调用 initSqlJs() (异步)
   │
2. 创建 Database 实例
   │
3. 执行 DDL (schema.ts)
   │   ├── 创建表
   ├── 创建索引
   ├── 创建 FTS4 虚拟表
   └── 创建触发器
   │
4. 执行迁移 (migrations/)
   │
5. 导出数据库到文件
   │
6. 完成初始化
```

#### 关键特性

- **同步操作**: 除 `initSqlJs()` 外，所有数据库操作同步执行
- **单例模式**: 数据库实例存储在模块作用域
- **FTS4 搜索**: 使用 FTS4 而非 FTS5 (sql.js WASM 限制)
- **持久化**: `db.export()` → `fs.writeFileSync()`

## 工具系统 (src/tools/)

### 工具设计模式

每个工具遵循统一的模式：

```typescript
export async function novel_tool_name(input: ToolInput, context: PluginContext): Promise<ToolOutput> {
  // 1. 检查数据库初始化
  const db = getDatabase();
  if (!db) {
    return {
      isError: true,
      content: [{ type: 'text', text: '请先初始化小说项目，使用 novel_init 工具' }],
    };
  }
  
  // 2. 解析输入参数
  const { param1, param2 } = input;
  
  // 3. 执行业务逻辑
  const result = doSomething(db, param1, param2);
  
  // 4. 返回结果
  return {
    content: [{ type: 'text', text: formatResult(result) }],
  };
}
```

### Agent 职责

#### World Builder (世界观构建师)

- 引导用户构建完整的世界观
- 力量体系设计
- 地理环境规划
- 文明结构设计
- 势力格局设定

#### Dungeon Master (副本主神)

- 副本主题设计
- 规则机制设计
- NPC 配置
- 奖励体系设计
- 攻略路线规划

#### Reviewer (网文审查员)

- 8 项质量标准检查
- AI 味检测
- 设定一致性检查
- 逻辑检查
- 自动修复建议

#### Plot Planner (大纲规划师)

- 篇章规划
- 节奏控制
- 伏笔设置
- 副本攻略路线规划
- 角色成长弧线设计

### 温度配置

| Agent | 温度 | 用途 |
|-------|------|------|
| PlotWriter | 0.85 | 创意写作 |
| WorldBuilder | 0.75 | 设定生成 |
| DungeonMaster | 0.75 | 副本设计 |
| PlotPlanner | 0.65 | 规划推理 |
| Reviewer | 0.25 | 严格审查 |
| DashboardGenerator | 0.80 | 面板生成 |
| 默认 | 0.70 | 通用 |
