### 调度器 (dispatcher.ts)

调度器是章节引擎的核心，负责将章节写作请求分解为多个场景并逐个调度。

#### 核心接口

```typescript
interface ChapterRequest {
  dungeonId: string;
  chapterNum: number;
  title: string;
  genre?: string;
  wordCount?: number;
}

interface SceneBlueprint {
  sceneNum: number;
  mood: string;
  pacing: string;
  sensoryFocus: string;
  tensionTechnique: string;
}

interface ChapterResult {
  content: string;
  scenes: SceneResult[];
  metadata: ChapterMetadata;
}
```

#### 调度流程

```
1. 接收章节请求
   │
2. 构建类型画像 (GenreProfilePack)
   │   ├── 加载类型模板
   ├── 解析风格规则
   └── 构建禁止/推荐模式
   │
3. 生成情感蓝图 (EmotionBlueprint)
   │   ├── 分析副本主题
   ├── 设计情绪曲线
   └── 划分场景节点
   │
4. 创建场景蓝图列表
   │   ├── Scene 1: 引入 (平静)
   ├── Scene 2: 冲突 (紧张)
   ├── Scene 3: 高潮 (激烈)
   └── Scene 4: 收尾 (缓和)
   │
5. 逐场景调度 (dispatchSceneWrite)
   │   ├── 加载场景上下文
   ├── 应用类型规则
   ├── 调用 LLM 写作
   └── 保存场景结果
   │
6. 组装完整章节
   │   ├── 合并场景内容
   ├── 生成 frontmatter
   └── 添加 wikilink
   │
7. 写回存储
   │   ├── 保存 Markdown 文件
   ├── 更新数据库
   └── 更新向量存储
   │
8. 返回结果
```

## 2. 审查系统 (src/modules/review/)

审查系统负责章节质量检查和 AI 味检测。

### 8 项标准检查

| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 禁用词扫描 | 检查禁用词汇 | blocker |
| 人称视角一致性 | 检查视角切换 | blocker |
| 模拟失忆泄露 | 检查信息泄露 | blocker |
| 段落结构 | 检查段落长度 | warning |
| 章尾检查 | 检查结尾处理 | warning |
| AI 味扫描 | 检查 AI 特征 | warning |
| 设定一致性 | 检查设定矛盾 | blocker |
| 逻辑检查 | 检查逻辑错误 | blocker |

### AI 味检测 (anti-ai-rules.ts)

AI 味检测是审查系统的特色功能，包含 60+ 检测模式。

#### 检测层级

| 层级 | 模式数 | 说明 |
|------|--------|------|
| adverb_overuse | 15+ | 副词滥用 |
| emotion_tagging | 10+ | 情绪标签 |
| dialog_formality | 8+ | 对话过于正式 |
| structure_closure | 10+ | 结构闭合过度 |
| transition_formula | 8+ | 过渡公式化 |
| summary_tendency | 5+ | 总结倾向 |
| info_exposition | 8+ | 信息 exposition |

#### 规则结构

```typescript
interface AntiAiRule {
  id: string;
  layer: string;
  pattern: RegExp;
  description: string;
  severity: 'low' | 'medium' | 'high';
  suggestion: string;
}

interface AntiAiChange {
  ruleId: string;
  original: string;
  suggested: string;
  position: [number, number];
}
```

#### 检测流程

```
1. 加载规则 (loadAntiAiRules)
   │
2. 逐层检测
   │   ├── adverb_overuse
   ├── emotion_tagging
   ├── dialog_formality
   ├── structure_closure
   ├── transition_formula
   ├── summary_tendency
   └── info_exposition
   │
3. 生成修改建议
   │
4. 返回检测结果
```

### 自动修复 (anti-ai-apply.ts)

自动修复功能根据检测结果应用修改建议。

```typescript
function applyAntiAiFix(text: string): FixResult {
  // 1. 检测问题
  const issues = detectAntiAiPatterns(text);
  
  // 2. 生成修改
  const changes = issues.map(issue => ({
    original: issue.match,
    replacement: issue.suggestion,
  }));
  
  // 3. 应用修改
  let fixed = text;
  for (const change of changes) {
    fixed = fixed.replace(change.original, change.replacement);
  }
  
  // 4. 返回结果
  return { fixed, changes };
}
```

## 3. 一致性检查 (src/modules/consistency/)

一致性检查确保跨章节的设定和角色状态保持一致。

### 检查维度

| 维度 | 说明 |
|------|------|
| 世界观一致性 | 设定不矛盾 |
| 角色状态一致性 | 能力、物品、关系一致 |
| 时间线一致性 | 事件顺序合理 |
| 地理一致性 | 地点描述一致 |
| 规则一致性 | 副本规则不冲突 |
